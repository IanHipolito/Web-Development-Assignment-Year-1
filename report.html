1. Problem Description

I made a five page website not including the contact form or report page about a well known football called called Paris Saint Germain or PSG for sort. The five pages include a Home page, an About page, # a page that showcases PSG's players on its first team, a page that showcases PSG's achievements throughout the years and its player's achievements also. All the pages include a responsive and interactive navbar and footer. The footer includes links to the other pages of my website and also some of PSG's offical websites. The content my website offers is an incite to PSG's history and how they progressed throughout the years and how they are at present day with the use of videos, images and written context. I used a lots of html, css and a good bit of javascript to make my website as appealling to the audience to the best of my ablility. With going through my website I can guarantee you will know so much about PSG. I also showcase the entire first squad with their names and jersey numbers. I really like PSG's kits so I made of the merchanise or kits to showcase their 2021/2022 kit drops in the case someone wanted to know what they looked like and purchased them.

The archetypical users of my website do not have to be major PSG fans or even football fans. It is user friendly and easy to understand for everyone. However, the majority of my users or people that would view my website would be those who are interested in sports, mainly football. Users who know about football may come to my site to gain some knowledge about PSG since they are interested or invested in the sport. My user audience would also include French people or native Parisians who want to know about their countries biggest football club.

When I was researching about PSG's foundation and history, there was not many useful resources out there about it. There were a few but that was about it. Most of the pages were either very old and did not have much information or were just generally hard to understand. That is why I think my website solves that problem as the information is fairly easy to take in and is very user friendly unlike most of the websites I went into when tyring to research for this topic. It also solves the problem of a lack of condensed information on my particular topic.

3. Technology Selection and Site Architecture

For my website i used the latest versions HTML5 and CSS3. I made the website mobile friendly. As you compress the website the various text and images shrink and this allows for a user friendly experience. I incoporated javascript to allow the nav bar to change into a hamburger menu when the page becomes a certain smaller size. Various paragraph text align to the centre and font size decrease as you move into mobile use. Size of images alternate as you switch into different screens and this allows users to have the same experience throughout different devices. I also used corousel slideshow using css and javascript which makes for a more appealing website allowing users to engage more with the content.

4. JavaScript Functionality

In total I added four pieces of working JavaScript to my website. The first being my navigation bar for each of my pages. I made my navigation bar responsive and can be used for people using a mobile device as when you shrink the page a burger menu will come up and the user will be able to press it to reveal all the other pages. I also made a carousel slideshow or rotator for my home page. For my carousel slideshow I used a javascript function where I set a timer and the next image appears after that timer and so on. When the state is on the last image I made sure to start the loop again making it continuous. For my about page I used a short javascript function that allows the user to scroll down the page and the video will fade away as you scroll down the page. My achievement page included a image slider that uses a javascript function. The javascript function allows the user to use arrows on either side of the slider to switch from the previous or next image. I also included dots which the user can click and each dot corresponds to a specfic image and will always go to that image when clicked. I also used javascript to validate three pieces of my contact form. The user cannot enter an invalid email, name or phone number and the user will not be able to submit a contact form.
